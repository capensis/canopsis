# Notes de version Canopsis 3.45.0

Canopsis 3.45.0 a été publié le 11 septembre 2020.

## Procédure d'installation

Suivre la [procédure d'installation de Canopsis](../guide-administration/installation/index.md).

## Procédure de mise à jour

Suivre la [procédure standard de mise à jour de Canopsis](../guide-administration/mise-a-jour/index.md).

## Changements entre Canopsis 3.44.1 et 3.45.0

### Liste des modifications

*  **Documentation :**
    * Ajout d'une [documentation des variables d'environnement](../guide-administration/administration-avancee/variables-environnement.md) ([#2577](https://git.canopsis.net/canopsis/canopsis/-/issues/2577))
    * Ajout d'une [documentation des limitations de Canopsis](../guide-utilisation/limitations/index.md) ([#2576](https://git.canopsis.net/canopsis/canopsis/-/issues/2576))
    * Ajout d'un [schema de l'enchaînement des moteurs](../guide-administration/moteurs/schema-enchainement-moteurs.md) ([#2561](https://git.canopsis.net/canopsis/canopsis/-/issues/2561))
*  **UI :**
    *   **Bac à alarmes**
        * Ajout des majuscules dans les popups d'informations ([#2549](https://git.canopsis.net/canopsis/canopsis/-/issues/2549))
        * Correcttion d'un bug qui ne donne pas le bon nombre d'alarme lié (Dans quel cas ?) ([#2479](https://git.canopsis.net/canopsis/canopsis/-/issues/2479))
    *   **Context explorer**
        * Correction d'un bug qui rendait impossible d'activer/désactiver un watcher ([#2322](https://git.canopsis.net/canopsis/canopsis/-/issues/2322))
    *   **Widget Météo de service**
        * Dans le widget météo, le texte est aligné à gauche désormais ([#2533](https://git.canopsis.net/canopsis/canopsis/-/issues/2533))
    *   **Explorateur de Contexte**
        * Il est possible de trier par webhook dans l'explorateur de contexte ([#2492](https://git.canopsis.net/canopsis/canopsis/-/issues/2492))
    *   **Explorateur de Contexte**
        * Il n'est désormais plus possible de supprimer des pbehavior sans droits ([#2491](https://git.canopsis.net/canopsis/canopsis/-/issues/2491))
    *   **UIv2**
        * L'UIv2 est désormais supprimé de Canopsis ([#2481](https://git.canopsis.net/canopsis/canopsis/-/issues/2481))
*  **Moteurs :**
    *  **Engine-watcher**
        * Correction de plusieurs bugs constatés dans le fonctionnement des watchers dans des conditions particulières ([#2543](https://git.canopsis.net/canopsis/canopsis/-/issues/2543))
    *  **Webserver**
        * Ajout d'une option `--limit-request-line` qu'il est possible de surcharger grâce à la variable d'environnement `LIMIT_REQUEST_LINE` ([#2525](https://git.canopsis.net/canopsis/canopsis/-/issues/2525))
    *  **Engine-axe**
        * Modification du code de façon à ce que l'id d'une alarme soit réellement unique ([#2547](https://git.canopsis.net/canopsis/canopsis/-/issues/2547))
    *  **Engine-action**
        * Ajout d'une priorité sur les règles ([#2531](https://git.canopsis.net/canopsis/canopsis/-/issues/2531))
    *  **Engine-correlation**
        * Le moteur correlation est désormais activé par défaut ([#2522](https://git.canopsis.net/canopsis/canopsis/-/issues/2522))
            * Vous pouvez retrouver la documention ici : https://doc.canopsis.net/guide-administration/moteurs/moteur-correlation/
    *  **Engine-fifo**
        * Correction d'un bug qui empêchait la gestion d'une alarme s'il y a un ":" ([#2579](https://git.canopsis.net/canopsis/canopsis/-/issues/2579))
    *  **Engine-che**
        * Correction d'un bug qui met en erreur de moteur che ([#2426](https://git.canopsis.net/canopsis/canopsis/-/issues/2426))
*  **Linkbuilder**
    * ([#2470](https://git.canopsis.net/canopsis/canopsis/-/issues/2470))
